# Swarms Improvement Proposal (SIP) Guidelines

A Swarms Improvement Proposal captures the motivation, design, tradeoffs, and rollout plan for meaningful framework changes.

## A good SIP should cover

- the problem being solved
- the proposed API or behavior change
- alternatives considered
- migration or compatibility impact
- open questions for maintainers and contributors

## Before you submit

1. Search existing issues and pull requests for related work.
2. Gather examples that show why the change is needed.
3. Keep the proposal narrow enough to review and implement incrementally.
