# MAKER

MAKER is a multi-step orchestration pattern for tasks that benefit from repeated sampling, review, and validation before each step is accepted.

## When to use MAKER

Use MAKER for workflows where precision matters more than raw speed, especially when each stage should be checked before the next stage runs.
